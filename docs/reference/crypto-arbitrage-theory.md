# 크로스 거래소 크립토 아비트리지 이론

## 1. 아비트리지란?

아비트리지(Arbitrage, 차익거래)는 **동일한 자산이 서로 다른 시장에서 다른 가격에 거래될 때**, 싼 곳에서 사서 비싼 곳에서 팔아 무위험 수익을 얻는 전략이다.

```
시장 A: BTC = $60,000
시장 B: BTC = $60,300

→ A에서 매수, B에서 매도 → $300 차익 (수수료 제외)
```

이론적으로 아비트리지는 **무위험(risk-free)** 이익이지만, 실제로는 실행 지연, 수수료, 슬리피지 등의 비용이 존재한다.

### 효율적 시장 가설과의 관계

효율적 시장 가설(EMH)에 따르면 모든 정보가 가격에 즉시 반영되어 아비트리지 기회는 존재하지 않아야 한다. 그러나 크립토 시장은:

- 24/7 운영되는 수백 개의 독립 거래소
- 거래소별 유동성과 참여자 구성 차이
- 법정화폐(KRW, USD 등) 간 환율 마찰

이러한 비효율성으로 인해 일시적 가격 괴리가 발생하며, 이것이 아비트리지 기회가 된다.

## 2. 아비트리지의 유형

### 2.1 단순 아비트리지 (Simple / Spatial Arbitrage)

두 거래소 간 동일 자산의 가격 차이를 이용한다.

```
업비트: ETH/KRW = 4,800,000원
바이낸스: ETH/USDT = 3,200 USDT
환율: 1 USDT = 1,480원 → 바이낸스 환산가 = 4,736,000원

스프레드 = (4,800,000 - 4,736,000) / 4,736,000 = +1.35%
```

**김치 프리미엄**이 대표적 사례다. 한국 거래소에서 해외보다 높은 가격이 형성되는 현상으로, 자본 이동 규제와 원화 유동성 특성에서 기인한다.

### 2.2 삼각 아비트리지 (Triangular Arbitrage)

단일 거래소 내에서 **3개 이상의 거래쌍을 순환**하여 차익을 얻는다.

```
시작: 1,000 USDT
  1) USDT → BTC  (BTC/USDT 매수)
  2) BTC  → ETH  (ETH/BTC 매도)
  3) ETH  → USDT (ETH/USDT 매도)
결과: 1,003 USDT → 0.3% 수익
```

순환 경로의 환율 곱이 1이 아닐 때 기회가 발생한다:

```
rate(A→B) × rate(B→C) × rate(C→A) > 1  →  양의 차익
```

### 2.3 크로스 거래소 삼각 아비트리지 (이 프로젝트)

**서로 다른 거래소 + 서로 다른 기축 통화**를 결합한 형태다.

```
업비트(KRW 기축) ←→ 바이낸스(USDT 기축)

경로 예시:
  KRW → BTC(업비트) → BTC 전송 → BTC→ETH(바이낸스) → ETH 전송 → ETH→KRW(업비트)
```

이 프로젝트에서는 실제 자산 이동 없이 **가격 모니터링과 기회 탐지**에 집중한다.

## 3. 수학적 모델

### 3.1 스프레드 계산

두 거래소 간 스프레드는 다음과 같이 계산한다:

```
스프레드(%) = (P_high - P_low) / P_low × 100
```

여기서 실행 가능한 스프레드를 구하려면 **호가(bid/ask)**를 사용해야 한다:

```
실행 가능 매수가 = ask (매도 호가) → 이 가격에 살 수 있음
실행 가능 매도가 = bid (매수 호가) → 이 가격에 팔 수 있음

실질 스프레드 = (bid_high - ask_low) / ask_low × 100
```

`last`(최근 체결가)로 계산하면 실제 실행 불가능한 환상적 스프레드가 나올 수 있다.

### 3.2 브릿지 코인과 내재 환율 (Implied Rate)

KRW 거래소와 USDT 거래소를 비교하려면 **환율 변환**이 필요하다. 특정 코인을 "브릿지"로 사용하면 **내재 KRW/USDT 환율**을 역산할 수 있다:

```
implied_rate(KRW/USDT) = price_krw(BTC) / price_usdt(BTC)

예: BTC 업비트 = 128,000,000 KRW, 바이낸스 = 85,000 USDT
→ implied_rate = 128,000,000 / 85,000 = 1,505.88 KRW/USDT
```

코인별 내재 환율이 서로 다르다면, 그 차이가 아비트리지 기회다.

```
implied_rate(BTC) = 1,505 KRW/USDT
implied_rate(ETH) = 1,498 KRW/USDT

→ ETH가 BTC 대비 상대적으로 저평가
→ ETH 매수(업비트) + BTC 매도(업비트) 조합으로 차익 가능
```

### 3.3 그래프 모델과 벨만-포드 알고리즘

아비트리지 탐지를 **그래프 문제**로 변환할 수 있다.

#### 그래프 구성

- **노드**: 각 거래소의 각 자산 (예: `upbit:BTC`, `binance:USDT`)
- **엣지**: 거래 가능한 변환 (거래쌍 또는 거래소 간 전송)
- **가중치**: 변환 비율의 **음의 로그**

```
weight = -log(exchange_rate)
```

#### 왜 음의 로그인가?

환율을 곱하는 문제를 덧셈 문제로 변환하기 위해서다:

```
rate(A→B) × rate(B→C) × rate(C→A) > 1

양변에 log를 취하면:
log(rate(A→B)) + log(rate(B→C)) + log(rate(C→A)) > 0

음수를 취하면:
-log(rate(A→B)) - log(rate(B→C)) - log(rate(C→A)) < 0

→ 음의 가중치 사이클 = 아비트리지 기회
```

#### 벨만-포드 알고리즘

벨만-포드는 **음의 가중치 사이클을 탐지**할 수 있는 최단 경로 알고리즘이다.

```
알고리즘:
1. 모든 노드 거리를 ∞로 초기화, 시작 노드는 0
2. |V|-1 번 모든 엣지에 대해 완화(relaxation) 수행:
   if dist[u] + weight(u,v) < dist[v]:
       dist[v] = dist[u] + weight(u,v)
3. 한 번 더 완화를 시도 → 갱신이 일어나면 음의 사이클 존재

시간 복잡도: O(V × E)
```

이 프로젝트에서:
- V = (거래소 수) × (코인 수) + (법정통화 노드)
- E = 거래쌍 수 + 거래소 간 전송 경로
- 10개 코인, 2개 거래소 기준: V ≈ 22, E ≈ 40~60 → 매우 빠름

## 4. 실제 비용과 제약

이론적 스프레드에서 다음 비용을 차감해야 실질 수익이 된다:

| 비용 항목 | 설명 | 일반적 범위 |
|-----------|------|-------------|
| 거래 수수료 | 메이커/테이커 수수료 | 0.04% ~ 0.25% |
| 출금 수수료 | 네트워크 전송 수수료 | 코인별 상이 |
| 슬리피지 | 주문량에 따른 가격 밀림 | 유동성에 따라 다름 |
| 전송 시간 | 블록 컨펌 대기 시간 | 1분 ~ 60분 |
| 환율 변동 | KRW/USD 환율 리스크 | 실시간 변동 |

### 손익분기 조건

```
실질 수익 = 스프레드(%) - 매수 수수료 - 매도 수수료 - 슬리피지 - 전송 비용

최소 스프레드 > 2 × 거래 수수료 + 전송 비용 + 슬리피지 마진
```

예시 (업비트 0.05% + 바이낸스 0.1%):
```
최소 스프레드 > 0.05% + 0.1% + α = 약 0.2% ~ 0.5% 이상이어야 유의미
```

## 5. 김치 프리미엄 (Korea Premium)

한국 거래소에서 크립토 가격이 해외보다 높게 형성되는 현상이다.

### 원인

1. **자본 이동 규제**: 외국환거래법에 의한 해외 송금 한도
2. **원화 마켓 격리**: KRW 직접 거래 → 글로벌 시장과 분리
3. **수급 불균형**: 국내 수요 > 글로벌 공급과의 연결성
4. **FOMO**: 상승장에서 국내 투자자들의 집중 매수

### 역프리미엄

하락장에서는 반대로 한국 가격이 해외보다 낮아지는 **역프리미엄**이 발생하기도 한다.

```
김치 프리미엄(%) = (업비트 가격 - 바이낸스 환산가) / 바이낸스 환산가 × 100

양수: 김치 프리미엄 (국내 > 해외)
음수: 역프리미엄 (국내 < 해외)
```

## 6. 이 프로젝트에서의 적용

이 프로젝트는 **실거래 없이 모니터링과 분석**에 집중한다:

1. **수집**: 업비트/바이낸스에서 10개 코인의 bid/ask/last/volume 실시간 수집
2. **스프레드 계산**: 브릿지 코인을 통한 내재 환율 및 코인별 스프레드 산출
3. **그래프 엔진**: 벨만-포드로 다중 경로 아비트리지 사이클 탐지
4. **대시보드**: 실시간 시각화 및 What-If 시뮬레이션 (수수료/슬리피지 조정)

## 참고 용어

| 용어 | 정의 |
|------|------|
| **Bid** | 매수 호가. 이 가격에 팔 수 있음 |
| **Ask** | 매도 호가. 이 가격에 살 수 있음 |
| **Spread** | Bid와 Ask의 차이 |
| **Slippage** | 예상 가격과 실제 체결 가격의 차이 |
| **Maker** | 호가창에 주문을 거는 사람 (유동성 제공) |
| **Taker** | 호가창의 기존 주문을 체결하는 사람 (유동성 소비) |
| **Order Book** | 현재 대기 중인 매수/매도 주문 목록 |
| **Implied Rate** | 자산 가격으로 역산한 내재 환율 |
| **Negative Cycle** | 그래프에서 가중치 합이 음수인 순환 경로 |
