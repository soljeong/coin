# 크립토 기초 개념 정리

## 1. 블록체인과 크립토 기초

### 블록체인이란?

분산된 컴퓨터 네트워크가 공동으로 유지하는 **변경 불가능한 거래 장부**다. 중앙 관리자 없이 참여자들이 합의(consensus)를 통해 거래를 검증한다.

### 코인 vs 토큰

| 구분 | 코인 (Coin) | 토큰 (Token) |
|------|-------------|--------------|
| 정의 | 자체 블록체인을 가진 암호화폐 | 다른 블록체인 위에서 동작 |
| 예시 | BTC, ETH, SOL | USDT(이더리움 위), UNI |
| 역할 | 네트워크 수수료 지불, 보안 유지 | 특정 서비스/프로토콜에서 사용 |

### 합의 메커니즘

블록체인이 "이 거래가 유효하다"고 동의하는 방법이다.

**작업증명 (Proof of Work, PoW)**
- 채굴자가 복잡한 수학 문제를 풀어 블록을 생성
- 막대한 전력 소모
- BTC가 대표적

**지분증명 (Proof of Stake, PoS)**
- 코인을 예치(스테이킹)한 검증자가 블록을 생성
- 에너지 효율적
- ETH(2022년 전환), SOL, ADA, DOT, AVAX가 대표적

## 2. 거래소 기초

### 중앙화 거래소 (CEX)

업비트, 바이낸스 같은 회사가 운영하는 거래소다. 사용자 자산을 거래소가 보관(수탁)한다.

**호가창 (Order Book)**

```
        매도 호가 (Ask)
  ┌──────────────────────┐
  │ 60,500  ×  0.3 BTC   │  ← 가장 싼 매도 (best ask)
  │ 60,600  ×  1.2 BTC   │
  │ 60,800  ×  0.5 BTC   │
  ├──────────────────────┤
  │ 60,400  ×  0.8 BTC   │  ← 가장 비싼 매수 (best bid)
  │ 60,300  ×  2.1 BTC   │
  │ 60,100  ×  0.4 BTC   │
  └──────────────────────┘
        매수 호가 (Bid)

스프레드 = best ask - best bid = 60,500 - 60,400 = 100
```

- **시장가 주문 (Market Order)**: 현재 호가에 즉시 체결. 빠르지만 슬리피지 발생 가능
- **지정가 주문 (Limit Order)**: 원하는 가격을 지정. 체결 보장 없음

**메이커 vs 테이커**

| 역할 | 행동 | 수수료 |
|------|------|--------|
| Maker | 호가창에 주문을 걸어 유동성 제공 | 보통 낮음 (0 ~ 0.1%) |
| Taker | 기존 주문을 가져가 유동성 소비 | 보통 높음 (0.04 ~ 0.25%) |

### 탈중앙화 거래소 (DEX)

스마트 컨트랙트로 운영되며 자산을 사용자가 직접 보관한다. Uniswap, PancakeSwap 등이 대표적이다.

**AMM (Automated Market Maker)**

호가창 대신 **유동성 풀**을 사용한다:

```
상수곱 공식: x × y = k

풀: 100 ETH × 200,000 USDT = 20,000,000 (= k)
1 ETH 매수 시:
  99 ETH × ? USDT = 20,000,000
  ? = 202,020 USDT
  비용 = 202,020 - 200,000 = 2,020 USDT/ETH
```

거래량이 클수록 가격 영향(price impact)이 커진다.

## 3. 스테이블코인

법정화폐 가치에 1:1로 고정(peg)되도록 설계된 크립토다.

| 유형 | 방식 | 예시 |
|------|------|------|
| 법정화폐 담보 | 은행에 실제 달러 보유 | USDT, USDC |
| 크립토 담보 | 다른 크립토를 초과 담보 | DAI |
| 알고리즘 | 알고리즘으로 공급량 조절 | (UST — 실패 사례) |

**USDT vs USDC**
- USDT (Tether): 가장 거래량이 많음. 담보 투명성 논란 있었음
- USDC (Circle): 규제 준수에 중점. 미국 규제 기관과 협력

이 프로젝트에서 바이낸스 기축은 USDT다.

## 4. 스테이킹 (Staking)

### 개념

PoS 블록체인에 코인을 예치하여 **네트워크 검증에 참여**하고 보상을 받는 것이다. 은행 예금의 이자와 유사하지만, 블록체인 보안에 기여하는 대가다.

```
ETH 32개 예치 → 검증자(Validator) 자격 획득
→ 블록 생성/검증 → 연 3~5% 보상 수령
```

### 직접 스테이킹 vs 위임 스테이킹

| 방식 | 설명 | 예시 |
|------|------|------|
| 직접 스테이킹 | 직접 검증 노드 운영 | ETH 32개 필요 |
| 위임 스테이킹 | 검증자에게 위임 | SOL, ADA, DOT |
| 거래소 스테이킹 | 거래소가 대신 스테이킹 | 업비트/바이낸스 제공 |

### 리퀴드 스테이킹 (Liquid Staking)

스테이킹하면 자산이 잠기는 문제를 해결한다. 예치하면 **파생 토큰**을 받아 다른 곳에서 활용할 수 있다.

```
ETH 예치 → stETH 수령 (Lido)
         → stETH를 DeFi에서 담보로 사용 가능
         → 스테이킹 보상도 계속 받음
```

### 언스테이킹 기간

대부분의 PoS 체인은 출금 시 **대기 기간**이 있다:

| 체인 | 언스테이킹 기간 |
|------|-----------------|
| ETH | 수일 ~ 수주 (출금 큐) |
| SOL | ~2일 (에포크 종료 시) |
| DOT | 28일 |
| ADA | 없음 (즉시 가능) |

### 슬래싱 (Slashing)

검증자가 악의적 행동(이중 서명 등)을 하면 예치금 일부를 몰수당한다. 네트워크 보안을 유지하는 경제적 인센티브다.

## 5. 런치풀 / 런치패드 (Launchpool / Launchpad)

### 런치패드 (Launchpad)

신규 토큰을 **할인된 가격에 미리 구매**할 수 있는 플랫폼이다.

```
프로젝트 X가 토큰 발행 예정
→ 바이낸스 런치패드에서 BNB 보유자 대상 추첨
→ 당첨자는 상장 전 가격(예: $0.10)에 매수
→ 상장 후 $0.50이면 5배 수익
```

- 참여에 거래소 토큰(BNB 등) 보유 필요
- 수요가 높아 추첨/배분 방식

### 런치풀 (Launchpool)

기존에 가진 코인을 **예치하면 신규 토큰을 무료로 받는** 방식이다. 원금 손실 없이(임시 예치) 신규 토큰을 파밍한다.

```
바이낸스 런치풀:
  BNB 100개 예치 (30일간)
  → 매일 신규 토큰 Y를 배분받음
  → 기간 종료 후 BNB 100개 회수 + 토큰 Y 보유
```

- 예치 기간 동안 BNB 가격 변동 리스크는 존재
- 바이낸스가 가장 활발하게 운영

### 에어드롭 (Airdrop)

프로젝트가 마케팅 또는 커뮤니티 보상 목적으로 **토큰을 무료 배포**하는 것이다.

```
유형:
  - 홀더 에어드롭: 특정 코인 보유자에게 자동 지급
  - 활동 에어드롭: 프로토콜 사용 이력 기반 (레트로액티브)
  - 소셜 에어드롭: SNS 팔로우/리트윗 조건

유명 사례:
  - Uniswap (2020): 사용자에게 UNI 400개 지급 → 당시 $1,200 가치
  - Arbitrum (2023): ARB 토큰 배포
```

## 6. 이자 농사 / 유동성 공급 (Yield Farming)

### 유동성 공급 (Liquidity Providing)

DEX의 유동성 풀에 자산 쌍을 예치하고 **거래 수수료 수익**을 받는다.

```
Uniswap ETH/USDT 풀에 공급:
  $5,000 상당 ETH + $5,000 USDT 예치
  → 해당 풀의 거래 수수료 0.3% 중 지분 비례 수령
```

### 비영구적 손실 (Impermanent Loss)

유동성 공급의 핵심 리스크다. 예치한 두 자산의 가격 비율이 변하면, **그냥 들고 있었을 때보다 손해**를 볼 수 있다.

```
초기: 1 ETH = 2,000 USDT, 1 ETH + 2,000 USDT 예치
ETH 가격 2배 상승 → 4,000 USDT

그냥 보유했다면: 1 ETH($4,000) + 2,000 USDT = $6,000
풀에 넣었다면:   0.707 ETH + 2,828 USDT = $5,657

비영구적 손실 = $6,000 - $5,657 = $343 (약 5.7%)
```

가격이 원래대로 돌아오면 손실이 사라지므로 "비영구적(impermanent)"이라 부른다.

### 이자 농사 (Yield Farming)

유동성 공급 + **추가 토큰 보상**을 합친 전략이다.

```
SushiSwap ETH/USDT 풀:
  거래 수수료 수익: 연 10%
  + SUSHI 토큰 보상: 연 30%
  = 총 연 40% (APY)
```

높은 APY는 보통 보상 토큰 가치 하락으로 지속되지 않는다.

## 7. DeFi 핵심 개념

### TVL (Total Value Locked)

DeFi 프로토콜에 예치된 총 자산 가치. 프로토콜의 규모를 나타내는 지표다.

### APR vs APY

| 지표 | 의미 | 계산 |
|------|------|------|
| APR | 단순 연이율 | 원금 × 이율 |
| APY | 복리 연이율 | 재투자 효과 포함 |

```
APR 12% (월 1%):
  APR 수익: $10,000 × 12% = $1,200
  APY 수익: $10,000 × (1.01^12 - 1) = $1,268

APY > APR (복리 효과)
```

DeFi에서 표시하는 높은 % 숫자가 APR인지 APY인지 반드시 확인해야 한다.

### 담보 대출 (Lending/Borrowing)

크립토를 담보로 맡기고 다른 자산을 빌리는 것이다. Aave, Compound가 대표적이다.

```
ETH 10개 담보 ($20,000 가치)
→ LTV 75% 기준 → 최대 $15,000 USDT 대출 가능
→ ETH 가격 하락 시 담보 비율 하락 → 청산(Liquidation) 위험
```

### 청산 (Liquidation)

담보 가치가 기준 이하로 떨어지면 자동으로 담보가 매각된다.

```
담보: 10 ETH (가격 $2,000 → $20,000)
대출: 15,000 USDT
청산 기준: LTV 90%

ETH → $1,700이면:
  담보 가치 = $17,000
  LTV = 15,000 / 17,000 = 88.2% → 아직 안전

ETH → $1,600이면:
  담보 가치 = $16,000
  LTV = 15,000 / 16,000 = 93.7% → 청산 발동!
```

## 8. 지갑과 자산 관리

### 핫월렛 vs 콜드월렛

| 구분 | 핫월렛 (Hot) | 콜드월렛 (Cold) |
|------|-------------|----------------|
| 인터넷 연결 | 항상 연결 | 오프라인 |
| 편의성 | 높음 | 낮음 |
| 보안 | 해킹 위험 | 매우 안전 |
| 예시 | MetaMask, 거래소 지갑 | Ledger, Trezor |

### 개인키와 시드구문

```
개인키 (Private Key): 자산을 제어하는 비밀번호
  → 절대 공유하지 말 것

시드구문 (Seed Phrase): 12~24개 단어로 된 지갑 복구 키
  → "abandon ability able about above absent ..."
  → 분실 시 자산 복구 불가
```

**"Not your keys, not your coins"** — 거래소에 보관하면 거래소가 해킹되거나 파산할 때 자산을 잃을 수 있다 (FTX 사태 등).

### 네트워크와 전송

같은 토큰이라도 **여러 네트워크**에서 존재할 수 있다:

```
USDT 전송 시 네트워크 선택:
  - ERC-20 (이더리움): 수수료 높음, 느림
  - TRC-20 (트론): 수수료 낮음, 빠름
  - BEP-20 (BSC): 수수료 낮음

주의: 받는 주소의 네트워크와 일치해야 함!
  ERC-20으로 보낸 USDT를 TRC-20 주소로 보내면 → 자산 소실
```

## 9. 시장 지표와 분석

### 시가총액 (Market Cap)

```
시가총액 = 현재 가격 × 유통 공급량

BTC: $60,000 × 19,600,000개 = $1.176T
```

### 거래량 (Volume)

24시간 동안 거래된 총 금액. 유동성의 대리 지표다. 거래량이 낮으면 슬리피지가 커진다.

### 공포 탐욕 지수 (Fear & Greed Index)

0~100 사이 값으로 시장 심리를 측정한다.

```
0~24:  극단적 공포 (Extreme Fear) → 매수 기회?
25~49: 공포 (Fear)
50~74: 탐욕 (Greed)
75~100: 극단적 탐욕 (Extreme Greed) → 과열?
```

### 도미넌스 (Dominance)

전체 크립토 시장에서 특정 코인이 차지하는 비율. BTC 도미넌스가 높으면 알트코인 약세 경향.

## 10. 규제와 세금 (한국 기준)

### 한국 규제 현황

- **특금법**: 가상자산 사업자 신고 의무
- **트래블룰**: 100만원 이상 전송 시 송수신인 정보 전달 의무
- **실명계좌**: KYC 인증된 원화 입출금 계좌 필요
- **해외 거래소**: 한국 거주자의 해외 거래소 이용은 가능하나, 원화 입출금 불가

### 과세

- 2025년부터 가상자산 양도차익 과세 시행 (기본공제 250만원, 세율 20%)
- 해외 거래소 소득도 신고 의무

### 아비트리지와 규제

거래소 간 자산 이동(특히 해외 거래소)은 **트래블룰**과 **외환 규제** 대상이 될 수 있다. 모니터링 용도의 이 프로젝트는 실거래를 수행하지 않으므로 직접적 규제 대상은 아니다.

| 용어 | 의미 |
|------|------|
| **HODL** | 장기 보유 (Hold의 오타에서 유래) |
